本リポジトリは、WPF アプリケーション開発において<br>
「ジェネリック型制約（where 制約）」を設計レベルで活用するためのサンプル実装をまとめたものです。<br> <br>
主眼は、UI テクニックや MVVM フレームワークの使い方ではありません。<br>
実行時エラーや if 分岐で場当たり的に防がれがちな問題を、<br>
型システムそのものに引き上げて解決する設計を示すことにあります。<br> <br>
---<br> <br>

## このリポジトリで扱うテーマ<br>

<br>
・ジェネリック型制約（where : T）を用いた設計の具体例<br>
・「データ表示」「入力検証」「コマンド／処理系」への横断的適用<br>
・Repository パターンと型制約の組み合わせ<br>
・CSV / JSON など異なる入力形式を型情報で制限する設計<br>
・実行時例外を MessageBox で知らせる前に、設計で潰す考え方<br>
<br>
---<br>
<br>
## 設計思想<br>
<br>
このサンプルでは、次の方針を一貫して採用しています。<br>
<br>
・if 文による分岐よりも、型制約による排除を優先する<br>
・「読めてはいけないデータ」は、そもそもコンパイル段階で読めなくする<br>
・Entity と Repository の責務を型で明確に分離する<br>
・ViewModel は「型の切り替え」だけを行い、View はそれに追従する<br>
<br>
設計上の意図がコードから読み取れることを重視しており、<br>
意図的に冗長に見える実装も含まれています。<br>
<br>
---<br>
<br>
## 特徴的な実装例<br>
<br>
・UserEntity は CSV 読み込み専用として定義されている<br>
・LogEntity は JSON 読み込み専用として定義されている<br>
・CsvRepository / JsonRepository は別クラスとして分離されている<br>
・Entity 生成ロジックは Repository 外部から factory として注入される<br>
・「CSV を読み込んだ」という事実を Token 型として表現する設計を採用している<br>
<br>
これにより、<br>
「型では保証されないはずの情報」を型として扱う例を示しています。<br>
